苹果高层：在 27 寸 iMac 使用触摸屏幕是一件荒谬的事

发布人： 张三 （2103481232）
时   间： 2016-11-28 14:25:31
附   件： 附件一 附件二

【Technews科技新报】苹果营销长 Phil Schiller 日前在一场专访中，谈及了一些老议题，包括 Mac 为何不搭载触摸面板，以及新款 MacBook Pro 上的 Touch Bar。

Schiller 指出，Mac 实际上与 Apple Watch、iPhone、iPad 一样，都是一种电脑，同时也各自根据一些原则，来为特定的消费需求服务。尽管没有仔细地说明它们各自的设计原则是什么，但 Schiller 认为对苹果来说，这些典范是不会进一步质问的。其中一个理由是，苹果过去的几次实验，让他们得知具有触摸屏幕的 Mac 其实对苹果生态系没有帮助。

Schiller 进一步谈到，苹果的观点是把产品当作一个平台。如果使用者在笔电上使用触摸屏幕，最后这也必须转移到桌上型电脑。然而，对用户来说，在 27 寸的 iMac 使用触摸屏幕，绝对是一场灾难。如果真的使用触摸屏幕的话，也表示苹果必须重新设计 macOS 的许多 UI，像是顶端的功能列表，如此一来，反而会伤害另一批习惯使用鼠标或触摸板的用户。因此，这终究来说是一种取舍：UX 设计师不能同时最佳化两种选择。一套 UI 不是只适合对按，就是只适合触摸。

值得一提的是，新款 MacBook Pro 的 Touch Bar 尽管内建了一枚独立的小处理器，不过 Schiller 也指出这不是试图把 iOS 塞进 Mac，只是在升级 Mac 的操作性。在过去，苹果已经把 iOS 的多点触摸拿进 Mac 的触摸板，而 Touch Bar 则是这个进程的延伸，用来提供“临时的”（part-time）的触摸体验，但又不会摧毁 25 年来苹果在笔电的基础设计。因此，Touch Bar 不是把 iOS 移植到 Mac，而只是想改进 Mac 本身。不过，Schiller 并没有多谈 Touch Bar 的长远目的究竟是什么。

至于 Touch Bar 的出现，是不是暗示苹果终将完全移除实体键盘？Schiller 对此则保守许多，仅指出苹果确实希望能让 Touch Bar 具有更多功能，可以一键完成更多事，但苹果目前还是会在键盘的基础上改进键盘，因此在短期内，让整片键盘完全变成触摸屏幕，应该还不是苹果的选择。

Schiller 总结谈话时则指出，任何一款笔电总是会有批评，因为在产品设计时必然会有所取舍。但“我相信我们的团队做了很聪明的选择，而 MacBook Pro 也是由优秀技术打造的最好的笔电”，Schiller 说。
